# 键盘字符检测项目后续操作完整说明

本文档用于指导你们从现在开始完成键盘字符检测项目，包括：拍什么图片、怎么拍、拍单个字符还是全键盘、如何命名、图片放哪里、如何标注、如何训练以及训练前检查。

## 1. 当前项目状态

目前你们已经完成：

1. Anaconda 环境配置；
2. YOLO / Ultralytics 安装；
3. 项目代码脚本生成；
4. 训练、预测、数据划分脚本可以运行帮助命令。

接下来重点不是继续安装软件，而是：

> 采集图片 → 筛选图片 → 标注图片 → 划分数据集 → 训练 YOLO → 测试检测效果。

## 2. 本项目到底要拍什么

### 2.1 不建议只拍单个字符

不建议只拍一个单独按键或一个单独字符，例如只拍一个 `A` 键、一个 `B` 键。

原因：

1. 项目最终目标是检测键盘图像中的多个字符或按键；
2. 单个字符图片太简单，和真实使用场景不一致；
3. YOLO 是目标检测模型，更适合在一张图中检测多个目标；
4. 如果只拍单个字符，后期在整张键盘图上效果可能不好。

### 2.2 推荐拍完整键盘和局部键盘

推荐采集以下几类图片：

1. 完整键盘直视图；
2. 键盘左半区域；
3. 键盘右半区域；
4. 数字键区域；
5. 功能键区域；
6. 字母主键盘区域；
7. 少量轻微偏移或轻微反光图片。

也就是说：

> 主要拍全键盘和局部键盘，不单独拍一个字符。

## 3. 拍摄方式要求

老师要求尽量直视拍照，所以本项目采用固定直视拍摄方案。

### 3.1 角度要求

拍摄时尽量满足：

1. 相机正对键盘；
2. 镜头平面尽量与键盘平面平行；
3. 不需要从左侧、右侧、低角度大量斜拍；
4. 可以有少量轻微偏移，但不能作为主要数据。

推荐：

- 手机或相机固定在键盘正上方；
- 使用手机支架、三脚架或临时支架；
- 键盘尽量摆正，不要明显倾斜。

### 3.2 距离要求

本项目暂时不把拍摄距离作为变量。

要求：

1. 拍摄距离基本固定；
2. 键盘在画面中的大小尽量一致；
3. 不要一张很近，一张很远；
4. 如果拍局部区域，可以移动键盘或调整画面，但仍保持正视。

### 3.3 清晰度要求

图片必须保证：

1. 字符能看清；
2. 键帽边界能看清；
3. 不要严重虚焦；
4. 不要严重运动模糊；
5. 不要严重反光导致字符不可见。

轻微模糊或轻微反光可以保留少量，用来增强模型鲁棒性。

## 4. 第一批图片采集计划

建议第一批采集 80 到 120 张。

推荐数量如下：

| 类型 | 建议数量 | 说明 |
| --- | ---: | --- |
| 完整键盘直视图 | 30 | 主要训练和展示使用 |
| 字母主键盘区域 | 15 | 包含 A-Z 大部分字母 |
| 左半部分键盘 | 10 | 包含 Q、W、A、S、Z、X 等 |
| 右半部分键盘 | 10 | 包含 O、P、K、L、M 等 |
| 数字键区域 | 10 | 包含 0-9 |
| 功能键区域 | 10 | 包含 Esc、Tab、Shift、Enter 等 |
| 不同光照直视图 | 20 | 正常、偏暗、偏亮、轻微反光 |
| 少量困难样本 | 5 到 10 | 轻微模糊、轻微阴影、轻微偏移 |

如果时间紧张，最少先采集 50 张也可以启动流程，但效果可能不稳定。

## 5. 拍摄场景建议

### 5.1 正常光照

这是主数据，占大部分。

要求：

1. 室内灯正常打开；
2. 键盘字符清晰；
3. 没有大面积阴影；
4. 没有严重反光。

### 5.2 较暗光照

少量采集。

目的：让模型适应较暗场景。

注意：不能暗到字符看不清。

### 5.3 较亮光照

少量采集。

目的：让模型适应亮度变化。

注意：不能过曝。

### 5.4 轻微反光

可以少量采集。

目的：提高鲁棒性。

注意：如果反光导致字符完全看不清，不建议用于训练。

### 5.5 轻微阴影

可以少量采集。

目的：模拟真实环境。

注意：阴影不要遮住大面积字符。

## 6. 图片应该放在哪里

### 6.1 原始图片目录

所有拍摄得到的原始图片先放入：

```text
vision/data/raw/
```

这个目录保存原始数据，不建议直接删除或覆盖。

### 6.2 训练图片目录

从原始图片中筛选清晰、有代表性的图片，复制到：

```text
vision/data/selected/
```

后续标注和训练主要使用 `selected` 中的图片。

### 6.3 标注文件目录

YOLO 标注文件放入：

```text
vision/data/labels/
```

图片和标注必须同名。

例如：

```text
vision/data/selected/20260425_kb01_front_0001.jpg
vision/data/labels/20260425_kb01_front_0001.txt
```

## 7. 图片命名规则

建议使用统一命名规则：

```text
日期_键盘编号_拍摄区域_场景_序号.jpg
```

例如：

```text
20260425_kb01_full_normal_0001.jpg
```

各字段含义：

| 字段 | 含义 | 示例 |
| --- | --- | --- |
| 日期 | 拍摄日期 | 20260425 |
| 键盘编号 | 第几个键盘 | kb01、kb02 |
| 拍摄区域 | 拍摄范围 | full、letter、left、right、num、func |
| 场景 | 光照或情况 | normal、dark、bright、reflect、shadow、blur |
| 序号 | 图片编号 | 0001、0002 |

建议统一使用以下英文代码：

| 区域代码 | 含义 |
| --- | --- |
| full | 完整键盘 |
| letter | 字母主键盘区域 |

| right | 右半部分键盘 |
| num | 数字键区域 |
| func | 功能键区域 |

## 10. 场景代码说明

建议统一使用以下英文代码：

| 场景代码 | 含义 |
| --- | --- |
| normal | 正常光照 |
| dark | 较暗光照 |
| bright | 较亮光照 |
| reflect | 轻微反光 |
| shadow | 轻微阴影 |
| blur | 轻微模糊 |
| offset | 轻微位置偏移 |

不要使用中文文件名，避免某些工具读取路径时出错。

## 11. 拍摄时的具体操作步骤

1. 清理键盘表面，保证字符可见；
2. 把键盘放在桌面上，尽量摆正；
3. 固定手机或相机，让镜头正对键盘；
4. 先拍 5 张完整键盘正常光照图片，检查清晰度；
5. 确认图片清晰后再批量拍摄；
6. 按照完整键盘、字母区、左半区、右半区、数字区、功能区依次采集；
7. 适当改变光照，例如开灯、关一部分灯、改变台灯方向；
8. 拍摄完成后全部复制到 `vision/data/raw/`；
9. 从 `raw` 中挑选训练用图片复制到 `vision/data/selected/`。

## 12. 哪些图片不要用于训练

以下图片不建议放入 `selected`：

1. 字符完全看不清；
2. 大面积过曝；
3. 严重反光；
4. 严重模糊；
5. 键盘只拍到很小一部分且无法判断字符；
6. 图片方向错误，例如倒置或旋转 90 度；
7. 文件名混乱、重复或带中文特殊字符。

可以保留在 `raw` 中，但不建议进入训练集。

## 13. 到底标注字符还是标注按键

本项目推荐：

> 标注整个按键，类别为按键上的主要字符或功能名。

例如：

1. 框住整个 `A` 键，类别为 `A`；
2. 框住整个 `1` 键，类别为 `1`；
3. 框住整个 `Enter` 键，类别为 `Enter`；
4. 框住整个空格键，类别为 `Space`。

不推荐只框字符笔画本身，因为字符太小，标注难度大，YOLO 小目标检测难度也更高。

## 14. 哪些按键需要标注

第一版建议标注以下类别：

1. 数字：`0-9`；
2. 字母：`A-Z`；
3. 常用功能键：`Esc`、`Tab`、`CapsLock`、`Shift`、`Ctrl`、`Alt`、`Enter`、`Backspace`、`Space`。

如果时间不够，可以先做简化版：

1. 数字：`0-9`；
2. 字母：`A-Z`。

功能键可以后续再加。

## 15. 标注文件命名要求

标注文件必须和图片同名。

正确示例：

```text
20260425_kb01_full_normal_0001.jpg
20260425_kb01_full_normal_0001.txt
```

错误示例：

```text
20260425_kb01_full_normal_0001.jpg
label_0001.txt
```

错误原因：脚本无法自动匹配图片和标签。

## 16. 标注格式要求

YOLO 标注文件每一行代表一个目标：

```text
class_id x_center y_center width height
```

示例：

```text
10 0.5123 0.3812 0.0421 0.0578
11 0.5610 0.3820 0.0415 0.0582
```

其中：

1. `class_id` 是类别编号；
2. `x_center` 是目标中心点横坐标；
3. `y_center` 是目标中心点纵坐标；
4. `width` 是目标框宽度；
5. `height` 是目标框高度；
6. 坐标都是相对于图片宽高的比例，范围为 0 到 1。

一般用 LabelImg、Roboflow 或 CVAT 标注时会自动生成这种格式，不需要手写。

## 17. 类别顺序必须和 dataset.yaml 一致

当前项目类别配置在：

```text
vision/dataset.yaml
```

里面已经定义了：

```text
0-9、A-Z、Esc、Tab、CapsLock、Shift、Ctrl、Alt、Enter、Backspace、Space
```

标注工具中的类别顺序必须和 `dataset.yaml` 保持一致，否则训练会出现类别错乱。

例如：

- `10` 对应 `A`；
- `11` 对应 `B`；
- `42` 对应 `Enter`；
- `44` 对应 `Space`。

## 18. 数据集划分

图片和标注准备好后，在项目根目录运行：

```bash
python vision/code/split_dataset.py
```

它会把数据整理到：

```text
vision/datasets/keyboard/
```

生成结构：

```text
vision/datasets/keyboard/
  images/
    train/
    val/
    test/
  labels/
    train/
    val/
    test/
```

默认比例为：

```text
train: 70%
val: 20%
test: 10%
```

## 19. 训练前检查清单

训练前必须检查：

1. `vision/data/selected/` 中有图片；
2. `vision/data/labels/` 中有对应 `.txt` 文件；
3. 图片和标签文件名完全一致；
4. 标注框都是框整个按键，而不是有时框字符、有时框键帽；
5. 类别名和 `dataset.yaml` 一致；
6. 已经运行 `python vision/code/split_dataset.py`；
7. `vision/datasets/keyboard/images/train` 中有训练图片；
8. `vision/datasets/keyboard/labels/train` 中有训练标签。

## 20. 开始训练

环境激活后，在项目根目录运行：

```bash
python vision/code/train_yolo.py --model yolov8n.pt --imgsz 640 --epochs 80 --batch 4
```

如果确认 GPU 可用，可以运行：

```bash
python vision/code/train_yolo.py --model yolov8n.pt --imgsz 640 --epochs 80 --batch 4 --device 0
```

如果显存不足，把 batch 调小：

```bash
python vision/code/train_yolo.py --model yolov8n.pt --imgsz 640 --epochs 80 --batch 2 --device 0
```

训练完成后，最佳模型会保存到：

```text
vision/models/best.pt
```

## 21. 预测测试

训练完成后运行：

```bash
python vision/code/predict_yolo.py --model vision/models/best.pt --source vision/data/raw
```

预测结果会保存到：

```text
vision/outputs/predict/
```

## 22. 常见错误

### 22.1 只有图片，没有标签

现象：运行 `split_dataset.py` 时出现 `skip without label`。

原因：图片没有对应的 `.txt` 标注文件。

解决：补充同名标注文件。

### 22.2 类别错乱

现象：明明是 `A`，检测结果显示 `B` 或其他类别。

原因：标注工具类别顺序和 `dataset.yaml` 不一致。

解决：统一类别顺序，重新检查标注。

### 22.3 训练后效果很差

可能原因：

1. 图片太少；
2. 标注框不准确；
3. 有漏标；
4. 有错标；
5. 光照太单一；
6. 字符太小；
7. 框选规则不统一。

优先解决数据和标注问题，再考虑调参数。

### 22.4 训练时报显存不足

现象：`CUDA out of memory`。

解决：降低 batch 或图片尺寸。

```bash
python vision/code/train_yolo.py --model yolov8n.pt --imgsz 640 --epochs 80 --batch 1 --device 0
```

## 23. 推荐你们现在立刻做的事情

按顺序完成：

1. 拍摄 80 到 120 张直视键盘图片；
2. 全部放入 `vision/data/raw/`；
3. 挑选清晰图片复制到 `vision/data/selected/`；
4. 使用 LabelImg、Roboflow 或 CVAT 标注整个按键；
5. 标注文件放入 `vision/data/labels/`；
6. 运行 `python vision/code/split_dataset.py`；
7. 运行 `python vision/code/train_yolo.py --model yolov8n.pt --imgsz 640 --epochs 80 --batch 4`；
8. 使用 `predict_yolo.py` 测试检测效果。

## 24. 最重要的原则

本项目最重要的不是模型参数，而是数据质量。

请务必做到：

1. 拍摄方式统一：直视为主；
2. 文件命名统一：英文、无空格、无中文；
3. 标注方式统一：框整个按键；
4. 类别顺序统一：和 `dataset.yaml` 一致；
5. 图片和标签同名：一张图片对应一个 `.txt`。

只要这几点做好，后续训练和答辩展示会顺利很多。
