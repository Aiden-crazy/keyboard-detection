# 原始图片采集规范

## 1. 采集目标

为键盘字符检测模型采集高质量、稳定的原始图片。本项目按照老师要求，以**直视拍照**为主，也就是相机基本正对键盘，尽量减少斜视角度变化。

本项目暂时不把拍摄距离作为主要变量，建议固定相机与键盘之间的距离，让图片中的键盘大小基本一致。这样可以降低项目难度，提高模型训练和演示的稳定性。

## 2. 拍摄设备

可以使用以下设备之一：

1. 手机摄像头；
2. USB 工业相机；
3. 笔记本摄像头；
4. HALCON 连接的相机采集模块。

如果你们已有 HALCON 平台，推荐优先使用 HALCON 连接相机采集图片。这样更符合机器视觉课程项目，也方便后续展示完整流程。

## 3. 图片分辨率建议

建议分辨率：

- 最低：1280 × 720；
- 推荐：1920 × 1080；
- 如果字符很小，可以使用更高分辨率。

注意：训练 YOLO 时通常会缩放到 640 或 960，但原始图片保留高分辨率有利于后续处理。

## 4. 拍摄内容

建议采集以下类型：

1. 完整键盘直视图；
2. 键盘左半部分直视图；
3. 键盘右半部分直视图；
4. 数字键区域直视图；
5. 功能键区域直视图；
6. 少量轻微角度偏差图片，用于提高鲁棒性；
7. 不同光照下的直视图片。

本项目不重点采集大角度斜视图，也不重点改变拍摄距离。

## 5. 光照变化

虽然角度和距离尽量固定，但光照仍建议适当变化，因为实际检测中光照会明显影响字符清晰度。

至少采集以下光照条件：

1. 正常室内灯光；
2. 较暗环境；
3. 较亮环境；
4. 轻微侧光；
5. 键帽轻微反光；
6. 局部轻微阴影。

不要采集太多严重反光或完全看不清字符的图片。

## 6. 拍摄角度和距离要求

本项目采用固定拍摄方式：

1. 相机尽量正对键盘；
2. 镜头平面尽量与键盘平面平行；
3. 键盘尽量完整出现在画面中；
4. 拍摄距离保持基本一致；
5. 允许少量轻微偏移，但不作为主要变量；
6. 不需要大量左斜、右斜、低角度图片。

推荐采集时在桌面上固定手机支架或相机支架，保证拍摄条件稳定。

## 7. 键盘类型

如果条件允许，采集不同键盘：

1. 黑色键盘；
2. 白色键盘；
3. 笔记本键盘；
4. 机械键盘；
5. 薄膜键盘；
6. 带背光键盘。

如果只有一个键盘，也可以完成项目。此时重点改变光照和键盘区域，而不是改变角度和距离。

## 8. 文件命名规则

建议命名：

```text
日期_键盘编号_场景编号_序号.jpg
```

示例：

```text
20260425_kb01_front_0001.jpg
20260425_kb01_dark_0002.jpg
20260425_kb01_leftpart_0003.jpg
```

## 9. 数据保存规则

1. 所有原始图片放入 `vision/data/raw/`；
2. 不要修改、压缩或覆盖原始图片；
3. 经过筛选准备用于训练的图片复制到 `vision/data/selected/`；
4. 标注文件放入 `vision/data/labels/`；
5. HALCON 采集或预处理输出放入 `vision/data/halcon/`。

## 10. 第一批采集任务

建议第一批先采集 80 到 120 张：

| 类型 | 数量 |
| --- | ---: |
| 完整键盘直视图 | 30 |
| 左半部分直视图 | 15 |
| 右半部分直视图 | 15 |
| 数字键/功能键区域 | 15 |
| 不同光照直视图 | 20 |
| 少量轻微偏移或轻微模糊 | 10 |

这样既符合老师“直视拍照即可”的要求，也保留了一定的光照变化，方便模型更稳定。

## 11. 注意事项

1. 字符必须尽量清晰可见；
2. 拍摄距离基本固定；
3. 拍摄角度以正视、直视为主；
4. 不需要大量斜视图片；
5. 键盘字符太小会影响训练效果；
6. 反光严重到看不清字符的图片不要作为主要训练数据；
7. 可以保留少量困难样本用于测试模型鲁棒性。
