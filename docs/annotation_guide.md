# 键盘字符标注规范

## 1. 标注目标

本项目采用目标检测方式标注键盘字符。每个可识别字符都用一个矩形框框住，并赋予对应类别。

例如：

- 字母键：`A`、`B`、`C`；
- 数字键：`0` 到 `9`；
- 功能键：`Esc`、`Tab`、`CapsLock`、`Shift`、`Ctrl`、`Alt`、`Enter`、`Backspace`、`Space`；
- 符号键：根据项目需要决定是否标注。

## 2. 推荐标注方式

### 方式 A：标注字符本身

只框住键帽上的字符图案区域。

优点：

- 检测结果直接对应字符；
- 框更精确。

缺点：

- 字符区域很小，标注难度高；
- 小目标检测难度较大。

### 方式 B：标注整个按键，类别为该按键字符

框住整个键帽，类别标为键帽上的字符或功能名。

优点：

- 标注更容易；
- 目标更大，YOLO 更容易检测；
- 对课程项目更友好。

缺点：

- 框的是按键，不是字符本体。

### 本项目建议

推荐使用方式 B：

> 框住整个按键，类别标为按键上的主要字符或功能名。

例如框住 `A` 键的整个键帽，类别为 `A`。

## 3. 标注工具建议

可以选择以下任意工具：

1. LabelImg：简单，适合 YOLO 格式；
2. Roboflow：网页标注方便，可自动划分数据集；
3. CVAT：功能强，适合多人标注；
4. HALCON 标注工具：如果已配置深度学习模块，也可使用。

初学建议使用 LabelImg 或 Roboflow。

## 4. YOLO 标注格式

每张图片对应一个同名 `.txt` 文件。

例如：

```text
image_0001.jpg
image_0001.txt
```

每一行格式为：

```text
class_id x_center y_center width height
```

其中坐标都是相对图片宽高归一化后的数值，范围是 0 到 1。

示例：

```text
0 0.5123 0.3812 0.0421 0.0578
1 0.5610 0.3820 0.0415 0.0582
```

## 6. 标注边界要求

1. 尽量框住整个键帽边界；
2. 不要框到相邻键；
3. 如果按键被严重遮挡，可以不标；
4. 如果按键只被轻微遮挡，但仍能判断类别，可以标；
5. 模糊到无法判断字符的按键不要标；
6. 同一类按键标注规则必须一致。

## 7. 特殊按键处理

建议统一名称：

| 键盘显示 | 类别名 |
| --- | --- |
| Esc | Esc |
| Tab | Tab |
| Caps Lock | CapsLock |
| Shift | Shift |
| Ctrl | Ctrl |
| Alt | Alt |
| Enter / Return | Enter |
| Backspace | Backspace |
| 空格 | Space |

## 8. 数据划分

建议划分为：

```text
train: 70%
val: 20%
test: 10%
```

注意：同一角度连续拍摄的相似图片不要全部放在训练集，应分散到验证集和测试集，避免测试结果虚高。

## 9. 质量检查清单

标注完成后检查：

1. 是否每张图片都有对应 `.txt` 标注文件；
2. 类别编号是否和 `dataset.yaml` 中类别顺序一致；
3. 是否有框偏移、漏标、错标；
4. 是否有中文类别名或空格类别名导致训练异常；
5. 是否把训练集和验证集路径配置正确。
