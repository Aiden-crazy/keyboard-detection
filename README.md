# 键盘字符检测机器视觉项目方案

## 1. 项目目标

本项目目标是对键盘图片中的按键字符进行检测与识别，输出每个字符的位置和类别，例如 `A`、`S`、`D`、`1`、`Enter`、`Shift` 等。

最终可以实现：

1. 输入一张键盘图片；
2. 检测出键盘上每个可见按键字符区域；
3. 标出字符类别和位置；
4. 生成检测结果图或检测结果文件。

## 2. 技术路线建议

### 结论

建议采用：

> 前期采集和图像预处理使用 HALCON，核心字符检测模型使用 YOLO，后期可选择把训练好的模型导入 HALCON 进行部署演示。

考虑到老师说尽量直视拍照即可，本项目不需要重点处理大角度斜视和距离变化。整体难度可以降低，数据采集和模型训练都会更稳定。

## 3. HALCON 和 YOLO 对比

| 方案 | 优点 | 缺点 | 适合情况 |
| --- | --- | --- | --- |
| HALCON 传统方法：阈值、形态学、模板匹配、OCR | 工业软件成熟；流程可视化；固定场景效果稳定；便于课堂展示 | 对光照、键盘型号、字符磨损仍然敏感；规则多；扩展性弱 | 摄像头固定、键盘固定、字符清晰、背景简单 |
| HALCON Deep Learning / Deep OCR | 可以在 HALCON 内完成深度学习流程；部署统一 | 学习成本较高；训练数据仍然需要准备；版本和许可可能受限 | 已熟悉 HALCON 深度学习模块 |
| YOLO 检测字符或按键 | 泛化能力强；资料多；适合不同光照和不同键盘 | 需要采集和标注数据；需要 Python/PyTorch 环境训练 | 课程项目、真实图片检测、需要较好效果 |
| YOLO 检测按键 + OCR 识别字符 | 类别数少，检测更稳；适合字符类别很多的情况 | 后处理复杂；OCR 对小字符可能不稳定 | 想扩展到更多键盘和字符 |

## 4. 推荐实现方案

### 推荐方案 A：YOLO 直接检测字符类别

把每个键盘字符当作一个目标，例如 `A`、`B`、`C`、`1`、`Esc`、`Enter` 都是类别。

优点：

- 实现直观；
- 标注完成后训练流程简单；
- 输出结果就是字符类别和位置；
- 适合课程设计和答辩展示。

缺点：

- 类别较多，数据量要相对多一些；
- `O/0`、`I/1`、小字体字符容易混淆。

### 推荐方案 B：YOLO 检测按键区域 + OCR/分类识别字符

第一步用 YOLO 检测每个按键区域，第二步裁剪按键图像，用分类模型或 OCR 识别字符。

优点：

- 检测类别少；
- 对后续扩展更灵活。

缺点：

- 项目流程更复杂；
- 需要写裁剪、排序、识别后处理。

### 本项目建议

如果你们时间有限，建议采用方案 A：

> 使用 YOLO 直接检测键盘按键类别，HALCON 用于采集、图像增强和最终演示。

标注时建议框住整个按键，类别标为该按键的主要字符或功能名。这样比只框字符本身更容易，检测目标也更大，模型更容易训练。

## 6. 数据量建议

最低可运行版本：

- 图片数量：80 到 120 张；
- 以完整键盘直视图和局部键盘直视图为主；
- 每类字符至少出现 20 到 30 次以上；
- 常用字符最好出现 50 次以上。

较好效果版本：

- 图片数量：200 到 500 张；
- 以直视拍照为主，拍摄距离基本固定；
- 主要覆盖不同光照、不同键盘区域、不同键盘型号；
- 少量加入轻微偏移、轻微模糊、轻微反光样本；
- 训练集、验证集、测试集比例约为 7:2:1。

## 7. 采集原则

1. 以直视拍照为主，角度和距离尽量固定；
2. 不要只拍完全一样的图片，可以改变光照和键盘局部区域；
3. 要包含清晰、轻微模糊、轻微反光等真实情况；
4. 原始图片必须保留，不要直接覆盖；
5. 文件名建议包含日期、键盘型号、场景和序号，例如：`20260425_kb01_front_0001.jpg`。

## 8. 推荐答辩展示流程

1. 展示采集的原始键盘图片；
2. 展示标注界面和标注结果；
3. 展示 YOLO 训练过程曲线；
4. 展示检测结果图；
5. 如果使用 HALCON，可展示 HALCON 对图片进行读取、预处理、调用模型或显示结果；
6. 对比正常光照、较暗光照、轻微反光等直视拍摄场景下的检测效果。

## 9. 下一步任务

1. 先采集 80 到 120 张直视原始图片放入 `data/raw/`；
2. 从中挑选清晰且有代表性的图片放入 `data/selected/`；
3. 确定需要识别的字符类别；
4. 使用 LabelImg、Roboflow、CVAT 或 HALCON 标注工具进行框选；
5. 使用 `code/` 文件夹中的脚本整理数据、训练和预测；
6. 根据效果继续补充光照变化或局部键盘图片。
